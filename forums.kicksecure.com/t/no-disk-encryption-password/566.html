Patrick September 9, 2024, 3:59pm 2

I don’t know how you fixed the double password entry requirement at boot bug, but congratulations.

The new upcoming ISO (version 17.2.2.0 and above) should have the double password entry bug fixed. (ISO: Change to unencrypted /boot if using Full Disk Encryption)

But I am unaware of any update which could fix it as this is something that the installer sets up. Perhaps if you moved from dracut to initramfs-tools, the double password entry issue might be resolved in earlier versions. Untested.

There is no need to report every change you observe. If there’s no issue, as per Support Request Policy, no need to report.

Reporting user observed changes is highly unlikely to ever help to spot malware if that is what you’re after. See also Valid Compromise Indicators versus Invalid Compromise Indicators.
