History for etc/calamares/modules/partition.conf - Kicksecure/live-config-dist

Contribute to Kicksecure/live-config-dist development by creating an account on GitHub.

ISO: Change to unencrypted /boot if using Full Disk Encryption Development
Calamares setting up encrpyted /boot (which gets decrypted with GRUB2) causes many issues. GRUB2 needs a very long time to decrypt the root disk. Need to enter full disk password twice, at grub boot menu and systemd during boot. This might be a missing dracut module or bug. And the worst issue: Keyboard layout issues. There is no way to define keyboard layout during GRUB2 pre-boot full disk encryption password entry. (If there is, these are very complicated, not suitable for a Debian derivativ…
Use Btrfs as the default journaling file system Development
Btrfs provide easy roll back snapshots compatible with timeshift or snapper. Good explanation about it: [BTRFS Guide | The Best Desktop File System] Atm it needs some manual work to make btrfs, good explanation can be found here: Many suggest to not use swap since you are on SSD e.g: [Debian 12 Bookworm Minimal Install w/BTRFS] [Debian 12: Fighting with OpenSUSE Snapper!] (Suggest to use zRAM instead, known to be even faster).
